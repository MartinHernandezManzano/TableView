El séptimo y último proyecto, TableView, cubre los ejercicios 23, 24, 25 y 26 y es el más completo del temario — combina JavaFX con JDBC y aplica la arquitectura MVC para construir una aplicación real de gestión de datos.
La arquitectura MVC divide el proyecto en tres capas con responsabilidades separadas. El Modelo (Empleado) es simplemente una clase Java que representa una fila de la tabla de base de datos — solo datos y sus getters, sin JavaFX ni SQL. El Controlador (Controlador) es quien habla con la base de datos mediante JDBC — aquí vive todo el SQL y devuelve listas de objetos Empleado a quien se los pida. La Vista (Vista) es la interfaz JavaFX — muestra los datos y reacciona a los eventos del usuario, pero no toca SQL en ningún momento. La clase Conexion es una utilidad compartida que centraliza la conexión a la base de datos leyendo los datos de acceso desde un fichero db.properties en lugar de tenerlos escritos en el código.
El ejercicio 23 es el punto de partida — conectar la aplicación a la base de datos y mostrar los empleados en un TableView. Cada columna de la tabla se vincula a un campo del modelo mediante PropertyValueFactory, que localiza automáticamente el getter correspondiente por nombre.
El ejercicio 24 amplía la aplicación con las operaciones CRUD completas. El Controlador añade los métodos insertar(), actualizar() y eliminar() usando PreparedStatement. La Vista añade un formulario debajo de la tabla con los campos necesarios y tres botones. Un detalle importante: al hacer clic en una fila de la tabla los campos del formulario se autocompletan solos gracias a un listener sobre la selección, lo que hace más cómodo editar o eliminar un empleado existente. Como la tabla EMPLEADO no genera el ID automáticamente, el controlador calcula el siguiente ID disponible con MAX(id)+1.
El ejercicio 25 añade un buscador en tiempo real encima de la tabla usando FilteredList — una lista especial que aplica un filtro dinámicamente cada vez que cambia su predicado. El buscador filtra por nombre sin distinguir mayúsculas de minúsculas, y se actualiza solo mientras el usuario escribe sin necesidad de pulsar ningún botón.
El ejercicio 26 ya estaba resuelto dentro del ejercicio 24 — el botón Eliminar selecciona el empleado marcado en la tabla, lo borra de la base de datos mediante JDBC y recarga la tabla para reflejar el cambio.
